Layer 1 — Core Logic Functions
🆕
New-VHDItem L262–417
Interactive 7-step wizard to create a new blank VHD/VHDX from scratch.
  • Step 1: Format selection (VHDX / VHD)
  • Step 2: Filename
  • Step 3: Size in GB
  • Step 4: Disk type (Dynamic / Fixed)
  • Step 5: Partition style (GPT / MBR)
  • Step 6: File system + compression + alloc unit
  • Step 7: Volume label → Confirmation → DiskPart
Creates VHD via DiskPart scripting. Each step renders on its own clear screen with cancel option.
📦
New-VHDCapsuleFromFolder L419–674
Packages an existing folder into a new VHD capsule with Robocopy transfer and verification.
  • Source folder validation
  • Auto-calculate size (source + headroom)
  • Configurable settings with review loop
  • DiskPart creation → Mount → Robocopy /E /MT:8
  • Post-copy verification report
  • Supports -Force for unattended execution
Largest function (256 lines). Combines VHD creation + data migration into one automated workflow.
🔧
Invoke-VHDManager L676–782
Operations hub for an existing VHD — mount, compact, defrag, clean, or enter Capsule Mode.
  • Contextual menu (mount/dismount toggle)
  • Current State analysis (size, files, fragmentation)
  • Compact via DiskPart (readonly attach)
  • Defragment via Optimize-Volume
  • Clean junk ($RECYCLE.BIN, SVI)
  • Gateway to Capsule Mode
Menu loop with state-aware UI — dismount option appears only when VHD is mounted.
💊
Invoke-CapsuleMode L784–915
Core innovation — isolated app execution with filesystem change tracking.
1. Mount → 2. Snapshot → 3. Execute → 4. Diff & Maintain
  • Pre-execution filesystem snapshot (all files)
  • Launch app → WaitForExit → 3s flush
  • Compare-Object diff on FullName + Time + Size
  • Post-run maintenance (compact, defrag, clean)
  • DirectLaunch mode → auto-dismount + exit
  • Default shortcut: launch_app.lnk
The "Capsule" concept: mount a VHD, run an app inside it, see exactly what changed, then maintain. Perfect for portable app isolation.
